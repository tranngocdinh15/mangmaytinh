Phụ thuộc hàm (Functional Dependency)
Nền tảng của lý thuyết chuẩn hóa

Định nghĩa phụ thuộc hàm

Cho quan hệ R(U). X, Y ⊆ U. Ta nói X → Y (X xác định hàm Y) nếu với mọi hai bộ t₁, t₂ trong R: t₁[X] = t₂[X] ⟹ t₁[Y] = t₂[Y]

Nghĩa là: biết giá trị của X thì xác định duy nhất giá trị của Y.

  • X gọi là vế trái (LHS) – thuộc tính xác định
  • Y gọi là vế phải (RHS) – thuộc tính phụ thuộc
  • Nếu X là siêu khóa thì X → U (X xác định tất cả)
Ví dụ: MaSV → HoTen, NgaySinh, Lop MaMon → TenMon, SoTiet (MaSV, MaMon) → Diem

Các loại phụ thuộc hàm

  • PTH đầy đủ: X → Y và không tồn tại X' ⊂ X mà X' → Y
  • PTH bộ phận: X → Y nhưng tồn tại X' ⊂ X mà X' → Y
  • PTH bắc cầu: X → Y → Z (Z phụ thuộc gián tiếp qua Y)
  • PTH hiển nhiên: X → Y khi Y ⊆ X

Hệ tiên đề Armstrong

  • Phản xạ: Y ⊆ X ⟹ X → Y
  • Tăng trưởng: X → Y ⟹ XZ → YZ
  • Bắc cầu: X → Y, Y → Z ⟹ X → Z
  • Hợp: X → Y, X → Z ⟹ X → YZ
  • Tách: X → YZ ⟹ X → Y và X → Z
Bao đóng của X dưới F: X⁺ = {A | X →_F A}

Thuật toán tính bao đóng X⁺

1. Khởi tạo: X⁺ = X 2. Lặp: Với mỗi PTH Y → Z trong F: Nếu Y ⊆ X⁺ thì X⁺ = X⁺ ∪ Z 3. Dừng khi X⁺ không thay đổi 4. Kết quả: X⁺ là bao đóng của X

X là siêu khóa ⟺ X⁺ = U (bao đóng bằng toàn bộ thuộc tính)

Các Dạng Chuẩn (Normal Forms)
1NF → 2NF → 3NF → BCNF

1NF Dạng chuẩn 1

Một quan hệ ở 1NF nếu tất cả các thuộc tính chỉ chứa giá trị nguyên tử (không phân chia được)
  • Không có thuộc tính đa trị
  • Không có thuộc tính phức hợp (lồng nhau)
  • Mỗi ô chứa đúng 1 giá trị
  • Tất cả quan hệ trong CSDL quan hệ đều phải ở 1NF
Ví dụ vi phạm 1NF
SinhVien(MaSV, HoTen, SoDT{nhiều số}) ← VI PHẠM SinhVien(MaSV, HoTen, SDT1, SDT2) ← Đúng 1NF

2NF Dạng chuẩn 2

Quan hệ R ở 2NF nếu: đạt 1NF và mọi thuộc tính không khóa phụ thuộc hàm ĐẦY ĐỦ vào khóa chính
  • Chỉ áp dụng khi khóa chính là khóa ghép
  • Loại bỏ phụ thuộc bộ phận vào khóa
  • Nếu khóa đơn: tự động đạt 2NF nếu đã ở 1NF
Ví dụ vi phạm 2NF
DIEM(MaSV, MaMon, TenMon, Diem) Khóa: (MaSV, MaMon) Vi phạm: MaMon → TenMon (bộ phận!) Tách thành: DIEM(MaSV, MaMon, Diem) MONHOC(MaMon, TenMon)

3NF Dạng chuẩn 3

Quan hệ R ở 3NF nếu: đạt 2NF và không có thuộc tính không khóa nào phụ thuộc bắc cầu vào khóa chính
  • Loại bỏ phụ thuộc bắc cầu
  • Định nghĩa chặt hơn: Với mọi X → A không hiển nhiên, hoặc X là siêu khóa, hoặc A là thuộc tính khóa
  • Bảo toàn phụ thuộc hàm khi phân rã
Ví dụ vi phạm 3NF
NHANVIEN(MaNV, MaPhong, TenPhong) Khóa: MaNV Vi phạm: MaNV → MaPhong → TenPhong (bắc cầu!) Tách thành: NHANVIEN(MaNV, MaPhong) PHONGBAN(MaPhong, TenPhong)

BCNF Boyce-Codd Normal Form

Quan hệ R ở BCNF nếu: với mọi phụ thuộc hàm X → Y không hiển nhiên, X phải là siêu khóa của R
  • Mạnh hơn 3NF – loại bỏ mọi dư thừa từ PTH
  • Phân rã BCNF không luôn bảo toàn phụ thuộc hàm
  • Nhưng luôn đảm bảo kết nối không mất mát
  • Mọi quan hệ ở BCNF đều ở 3NF, ngược lại chưa chắc
Ví dụ vi phạm BCNF
DKMON(SinhVien, MaMon, GiaoVien) Khóa: (SinhVien, MaMon) hoặc (SinhVien, GiaoVien) PTH: GiaoVien → MaMon Vi phạm: GiaoVien không là siêu khóa! Tách thành: GV_MON(GiaoVien, MaMon) SV_GV(SinhVien, GiaoVien)

Phân rã không mất mát (Lossless Join)

Phân rã R thành R₁, R₂ là không mất mát nếu: R = R₁ ⋈ R₂ (kết nối tự nhiên khôi phục lại R gốc)

Điều kiện (Định lý Heath):

Phân rã R thành R₁(XY) và R₂(XZ) là không mất mát ⟺ X → Y (∈ F⁺) HOẶC X → Z (∈ F⁺) Tức là: thuộc tính chung X phải là siêu khóa của ít nhất một trong hai quan hệ con.

Phân rã bảo toàn phụ thuộc hàm

Phân rã R(F) thành {R₁, R₂, ..., Rₙ} bảo toàn PTH nếu: F⁺ = (F₁ ∪ F₂ ∪ ... ∪ Fₙ)⁺ với Fᵢ = πRᵢ(F)

Kiểm tra bằng cách: tính lại bao đóng của các PTH trên từng quan hệ con và xem có suy dẫn lại được toàn bộ F không.

Thuật toán kiểm tra bảo toàn PTH: Với mỗi X → Y trong F: 1. Z = X 2. Với mỗi Rᵢ: Z = Z ∪ ((Z ∩ Rᵢ)⁺ ∩ Rᵢ) [lặp] 3. Nếu Y ⊆ Z thì X → Y được bảo toàn

Thuật toán phân rã về 3NF (Synthesis)

Bước 1: Tìm phủ tối tiểu Fc của F a) Đơn giản hóa vế phải: X → YZ thành X → Y, X → Z b) Loại thuộc tính thừa ở vế trái c) Loại PTH dư thừa Bước 2: Với mỗi X → Y trong Fc, tạo quan hệ Rᵢ(XY) Bước 3: Nếu không có Rᵢ chứa khóa của R, thêm Rk(K) Kết quả: đảm bảo 3NF + bảo toàn PTH + không mất mát

Thuật toán phân rã về BCNF

Bước 1: Bắt đầu với ρ = {R} Bước 2: Với mỗi Rᵢ ∈ ρ chưa ở BCNF: - Tìm X → A vi phạm BCNF (X không là siêu khóa) - Tách: R₁ = X⁺ (bao đóng của X trong Rᵢ) R₂ = Rᵢ - X⁺ + X (thuộc tính còn lại + X) - Thay Rᵢ bằng {R₁, R₂} Bước 3: Lặp đến khi tất cả đều ở BCNF Lưu ý: Có thể không bảo toàn PTH!

Tìm phủ tối tiểu (Canonical Cover)

Bước 1: Tách vế phải thành đơn F = {X → A | A là thuộc tính đơn} Bước 2: Loại thuộc tính thừa vế trái Với X → A, kiểm tra từng B ∈ X: Nếu A ∈ (X-{B})⁺ thì loại B khỏi X Bước 3: Loại PTH dư thừa Với X → A, kiểm tra: Nếu A ∈ (F - {X→A})* suy dẫn được từ F còn lại thì loại X → A

1
Cho quan hệ R(A,B,C,D,E) với tập phụ thuộc hàm F sau. Tìm tất cả các khóa của R.
F = { AB → C, B → D, CD → E, CE → A }

Lời giải

Tìm khóa bằng cách kiểm tra bao đóng. Nhận xét: B và E không xuất hiện ở vế phải → B, E có thể có trong khóa.

Kiểm tra (AB)⁺: AB → C (dùng AB→C): {A,B,C} B → D (dùng B→D): {A,B,C,D} CD → E (dùng CD→E): {A,B,C,D,E} = U ✓ → AB là siêu khóa. Kiểm tra A⁺, B⁺: A⁺ = {A} ≠ U B⁺ = {B,D} ≠ U → AB không giảm được → AB là khóa ✓ Kiểm tra (BE)⁺: CE → A: cần C trước → không dùng được BE⁺ = {B,D,E} ≠ U → không phải khóa Các tổ hợp khác: kiểm tra ABx: AB đã là khóa tối tiểu
  • Khóa duy nhất của R là: AB
2
Cho quan hệ R(MaSV, MaMon, TenMon, GiaoVien, Diem) với F sau. Xác định R thuộc dạng chuẩn nào và phân rã về 3NF.
F = { (MaSV, MaMon) → Diem, MaMon → TenMon, MaMon → GiaoVien } Khóa chính: (MaSV, MaMon)

Lời giải

Kiểm tra dạng chuẩn:

  • 1NF: ✓ (tất cả giá trị nguyên tử)
  • 2NF: ✗ Vi phạm! MaMon → TenMon và MaMon → GiaoVien là PTH bộ phận vào khóa ghép

Phân rã về 3NF:

R₁(MaSV, MaMon, Diem) Khóa: (MaSV, MaMon) PTH: (MaSV,MaMon) → Diem ← đầy đủ ✓ R₂(MaMon, TenMon, GiaoVien) Khóa: MaMon PTH: MaMon → TenMon, MaMon → GiaoVien ✓ Kết quả: Cả R₁ và R₂ đều ở 3NF Phân rã bảo toàn PTH ✓ Không mất mát (MaMon là thuộc tính chung, MaMon là khóa R₂) ✓
3
Tìm phủ tối tiểu của tập phụ thuộc hàm F sau.
F = { A → BC, B → C, A → B, AB → C }

Lời giải

Bước 1: Tách vế phải đơn:

F = { A→B, A→C, B→C, AB→C }

Bước 2: Loại thuộc tính thừa vế trái:

Xét AB→C: Thử bỏ A: B→C đã có trong F → C ∈ {B}⁺ → Bỏ A, giữ B→C (đã có sẵn) → AB→C dư thừa

Bước 3: Loại PTH dư thừa:

F = { A→B, A→C, B→C, AB→C(đã thành B→C) } Xét A→C: Từ A→B, B→C suy ra A→C bằng bắc cầu → A→C dư thừa, loại bỏ! Xét AB→C: đã xử lý ở bước 2 Phủ tối tiểu Fc = { A→B, B→C }
  • Kiểm tra: A⁺ = {A,B,C} = cùng kết quả với F gốc ✓
4
Cho R(A,B,C,D) với F = {A→B, B→C, C→D}. Kiểm tra phân rã sau có không mất mát không?
ρ = { R₁(A,B), R₂(B,C), R₃(C,D) }

Lời giải – Dùng thuật toán bảng (Chase Algorithm)

Lập bảng ban đầu: A B C D R₁: a b c₁ d₁ R₂: a₂ b c d₂ R₃: a₃ b₃ c d Áp dụng A→B: R₁ có a ở A → R₂ cũng phải có b ở B → R₂: (a₂, b, c, d₂) Áp dụng B→C: R₁(a,b,c₁), R₂(a₂,b,c,d₂) cùng b → đồng nhất C → R₁: (a, b, c, d₁) [c₁ = c] Áp dụng C→D: R₁(a,b,c,d₁), R₂(a₂,b,c,d₂), R₃(a₃,b₃,c,d) cùng c → đồng nhất D → R₁: (a, b, c, d) ← Hàng có toàn biến không có chỉ số! Kết luận: Xuất hiện hàng (a,b,c,d) → Phân rã KHÔNG MẤT MÁT ✓
5
Cho R(A,B,C,D) với F = {AB→C, C→D, D→A}. Phân rã R về BCNF.
F = { AB→C, C→D, D→A } Tìm khóa, kiểm tra BCNF, và phân rã nếu vi phạm.

Lời giải

Bước 1 – Tìm khóa:

B không xuất hiện ở vế phải → B chắc chắn trong khóa (AB)⁺: AB→C: {A,B,C}; C→D: {A,B,C,D} = U ✓ → Khóa là AB (kiểm tra A⁺={A,D} ≠ U, B⁺={B} ≠ U → AB tối tiểu)

Bước 2 – Kiểm tra BCNF:

C→D: C⁺ = {C,D,A} ≠ U → C không là siêu khóa → Vi phạm BCNF! D→A: D⁺ = {D,A} ≠ U → D không là siêu khóa → Vi phạm BCNF!

Bước 3 – Phân rã:

Xét C→D vi phạm: R₁ = C⁺ = {C,D,A} → R₁(A,C,D) R₂ = {C,D,A} - C⁺ + C = {B,C} → R₂(B,C) (vì R - {D,A} + C = {B,C}) Kiểm tra R₁(A,C,D) với F₁ = {C→D, D→A}: C⁺ = {C,D,A} = R₁ → C là siêu khóa ✓ BCNF Kiểm tra R₂(B,C) với F₂ = {AB→C (chiếu xuống BC = B→C?)} Không có PTH vi phạm → BCNF ✓ Kết quả: ρ = { R₁(A,C,D), R₂(B,C) } Lưu ý: AB→C bị mất (không bảo toàn PTH)
