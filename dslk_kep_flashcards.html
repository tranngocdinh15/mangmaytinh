1

Định Nghĩa DSLK Kép

Doubly Linked List (DLL) là danh sách liên kết mà mỗi node có 2 con trỏ:

  • pNext: trỏ đến node KẾ TIẾP
  • pPrev: trỏ đến node TRƯỚC ĐÓ
Mẹo Nhớ

"Hai chiều, hai con trỏ!" - Khác với DSLK đơn chỉ có 1 chiều, DSLK kép đi được CẢ 2 CHIỀU như đường 2 chiều!

2

Cấu Trúc NODE

typedef struct NODE { int info; // Dữ liệu NODE* pNext; // Con trỏ NEXT NODE* pPrev; // Con trỏ PREV } NODE;
Điểm Quan Trọng

Luôn có 3 thành phần: info + pNext + pPrev. Đây là câu hỏi trắc nghiệm PHỔ BIẾN!

3

Node Đầu & Node Cuối

Node Đầu (Head):

  • pPrev = NULL ✅

Node Cuối (Tail):

  • pNext = NULL ✅
Mẹo Nhớ

"Đầu không có PREV, Cuối không có NEXT" - Giống như tàu hỏa: đầu tàu không có toa trước, cuối tàu không có toa sau!

4

So Sánh DSLK Đơn vs Kép

Tiêu chí DSLK Đơn DSLK Kép
Con trỏ/node 1 (Next) 2 (Next, Prev)
Bộ nhớ Ít hơn Nhiều hơn
Duyệt ngược ❌ Không ✅ Được
Câu Hỏi Hay Gặp

"DSLK kép tốn bộ nhớ hơn nhưng linh hoạt hơn" - Đây là trade-off cần nhớ!

6

Thêm Vào Đầu (Insert Head)

Các bước:

  1. Tạo node mới p
  2. p→pNext = list.pHead
  3. p→pPrev = NULL
  4. Nếu pHead ≠ NULL: pHead→pPrev = p
  5. list.pHead = p
Mẹo Nhớ

"Next rồi Prev, đừng quên Head cũ!" - Phải cập nhật pPrev của Head CŨ trước khi gán Head MỚI!

7

Thêm Vào Cuối (Insert Last)

Các bước:

  1. Tạo node mới p
  2. Nếu danh sách rỗng: pHead = p
  3. Ngược lại: duyệt đến node cuối last
  4. last→pNext = p
  5. p→pPrev = last
  6. p→pNext = NULL
Dễ Nhầm

Nhớ kiểm tra danh sách rỗng trước! Nếu rỗng thì node mới CHÍNH LÀ head luôn.

8

Thêm Sau Node X (Insert After)

Cho trước: prev_node và new_data

Các bước:

  1. Kiểm tra prev_node ≠ NULL
  2. Tạo node mới p
  3. p→pNext = prev_node→pNext
  4. p→pPrev = prev_node
  5. prev_node→pNext = p
  6. Nếu p→pNext ≠ NULL: p→pNext→pPrev = p
Thứ Tự Quan Trọng

Luôn gán Next trước, Prev sau! Tránh mất liên kết!

9

Thêm Trước Node X (Insert Before)

Các bước:

  1. Kiểm tra next_node ≠ NULL
  2. Tạo node mới p
  3. p→pPrev = next_node→pPrev
  4. p→pNext = next_node
  5. next_node→pPrev = p
  6. Nếu p→pPrev ≠ NULL: p→pPrev→pNext = p
  7. Ngược lại: pHead = p (node mới là head)
Trường Hợp Đặc Biệt

Nếu thêm trước HEAD → node mới TRỞ THÀNH head mới!

10

Xóa Node

Xóa node p:

  1. Nếu p→pPrev ≠ NULL: p→pPrev→pNext = p→pNext
  2. Ngược lại: pHead = p→pNext
  3. Nếu p→pNext ≠ NULL: p→pNext→pPrev = p→pPrev
  4. delete p;
Mẹo Nhớ

"2 hướng, 2 cập nhật!" - Phải cập nhật CẢ 2 con trỏ của 2 node láng giềng!

Câu Hỏi Thi

Xóa node ở giữa danh sách, cần cập nhật mấy con trỏ?

Đáp án: 4 con trỏ (pNext của node trước, pPrev của node sau, và 2 con trỏ của node bị xóa)

11

Duyệt Danh Sách

Duyệt Xuôi (từ Head):

NODE* p = list.pHead; while (p != NULL) { // Xử lý p p = p->pNext; }

Duyệt Ngược (từ Tail):

NODE* p = tail; while (p != NULL) { // Xử lý p p = p->pPrev; // Dùng pPrev! }
Ưu Điểm DSLK Kép

Có thể duyệt 2 CHIỀU - điều mà DSLK đơn KHÔNG làm được!

12

Khởi Tạo Node

NODE* CreateNode(int x) { NODE* p = new NODE; if (p != NULL) { p->info = x; p->pNext = NULL; p->pPrev = NULL; } return p; }
Chú Ý

Luôn khởi tạo CẢ 2 con trỏ = NULL để tránh lỗi!

13

Khởi Tạo Danh Sách

typedef struct LIST { NODE* pHead; } LIST; void Initialize(LIST& list) { list.pHead = NULL; } bool IsEmpty(LIST list) { return (list.pHead == NULL); }
14

Tìm Kiếm Node

NODE* Search(LIST list, int x) { NODE* p = list.pHead; while (p != NULL) { if (p->info == x) break; p = p->pNext; } return p; // NULL nếu không tìm thấy }
Lưu Ý

Duyệt bằng pNext, không dùng pPrev khi tìm từ đầu!

15

Insert Head - Code Đầy Đủ

void insertHead(LIST& list, int new_data) { NODE* p = CreateNode(new_data); p->pNext = list.pHead; if (list.pHead != NULL) list.pHead->pPrev = p; list.pHead = p; }
Câu Hỏi Thi

Nếu quên dòng if (list.pHead != NULL) thì sao?

Đáp án: Lỗi khi danh sách RỖNG (NULL pointer dereference)!

16

Insert After - Code Đầy Đủ

void insertAfter(NODE* prev_node, int new_data) { if (prev_node == NULL) { printf("Cannot insert after NULL"); return; } NODE* p = CreateNode(new_data); p->pNext = prev_node->pNext; p->pPrev = prev_node; prev_node->pNext = p; if (p->pNext != NULL) p->pNext->pPrev = p; }
Mẹo Ghi Nhớ Thứ Tự

"Next của mới → Next của cũ" rồi mới cập nhật ngược lại!

17

Insert Last - Code Đầy Đủ

void insertLast(LIST& list, int new_data) { NODE* p = CreateNode(new_data); if (list.pHead == NULL) { list.pHead = p; return; } NODE* last = list.pHead; while (last->pNext != NULL) last = last->pNext; last->pNext = p; p->pPrev = last; }
Độ Phức Tạp

O(n) - Phải duyệt hết danh sách để tìm node cuối!

⭐ Câu Hỏi Hay Gặp #1

Câu Hỏi

DSLK kép khác DSLK đơn ở điểm nào?

  • A. Có 2 con trỏ pNext
  • B. Có thể duyệt 2 chiều
  • C. Nhanh hơn DSLK đơn
  • D. Tốn ít bộ nhớ hơn
Đáp Án Đúng

B. Có thể duyệt 2 chiều

Giải thích: DSLK kép có pNext VÀ pPrev nên duyệt được cả xuôi lẫn ngược!

19

⭐ Câu Hỏi Hay Gặp #2

Câu Hỏi

Khi xóa node ở giữa DSLK kép, cần cập nhật mấy liên kết?

  • A. 1 liên kết
  • B. 2 liên kết
  • C. 3 liên kết
  • D. 4 liên kết
Đáp Án Đúng

B. 2 liên kết

Cập nhật: pNext của node trước + pPrev của node sau!

20

⭐ Câu Hỏi Hay Gặp #3

Câu Hỏi

Thêm node vào đầu DSLK kép, phải làm gì TRƯỚC TIÊN?

  • A. Cập nhật pHead
  • B. Gán pNext của node mới
  • C. Gán pPrev của node mới = NULL
  • D. Kiểm tra danh sách rỗng
Đáp Án Đúng

B. Gán pNext của node mới

Thứ tự: pNext của mới → pPrev của cũ → cập nhật Head!

21

💡 Mẹo Nhớ Tổng Hợp

5 ĐIỂM VÀNG CẦN NHỚ:

  1. 2 chiều = 2 con trỏ (pNext + pPrev)
  2. Head.pPrev = NULL, Tail.pNext = NULL
  3. Thêm/Xóa: cập nhật CẢ 2 CHIỀU!
  4. Kiểm tra NULL trước khi truy xuất!
  5. Tốn bộ nhớ NHƯNG linh hoạt hơn DSLK đơn!
Khẩu Quyết Thi

"Next - Prev - NULL - Check - Update!"

Lưu Ý Quan Trọng

Xóa node trong DSLK kép nhanh hơn DSLK đơn vì không cần tìm node trước đó!

TOP 5 LỖI KHI LÀM BÀI THI:

  2. ❌ Không kiểm tra NULL trước khi truy xuất

  4. ❌ Quên case danh sách rỗng
  5. ❌ Nhầm giữa pNext và pPrev khi duyệt
